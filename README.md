# ShuttleQueue | TheBreakfastClub Badminton Manager

A professional-grade badminton club management application built with **Next.js 16**, **Tailwind CSS**, **Firebase**, **Supabase**, and **Genkit AI**.

## 🏗️ Architecture

### **Hybrid Data Layer**
- **Local Storage (Primary)**: Fast UI experience with immediate data updates
- **Firebase Firestore (Backend)**: Reliable cloud storage with real-time sync
- **Automatic Sync**: Background synchronization between local storage and Firebase
- **Offline Support**: Works offline with automatic sync when connection restored

### **Image Storage**
- **Supabase Storage**: Free tier for QR codes and club logos
- **Automatic Compression**: Images compressed before upload to save bandwidth

## 🚀 Quick Start & Setup

### **1. Firebase Project Connection**
The app is configured to connect to:
- **Project ID**: `studio-8289009920-31c2b`

**Firebase Services Used:**
- **Firestore** - Backend database for sessions, players, matches, and rankings
- **Firebase Authentication** - User authentication and session management
- **Firebase Cloud Messaging (FCM)** - Push notifications for court assignments
- **Realtime Database** - Real-time data synchronization
- **Remote Config** - Remote configuration management

**Supabase Services Used:**
- **Storage** - Free tier image storage for QR codes and club logos

### **2. Environment Variables**
Create a `.env.local` file in the project root with the following variables:

```env
# Firebase Admin SDK (for server-side notifications)
GOOGLE_APPLICATION_CREDENTIALS=D:\Dev\TheBreakfastClub-Badminton-App\firebase-admin-key.json

# Firebase VAPID Key (for web push notifications)
NEXT_PUBLIC_FIREBASE_VAPID_KEY=YOUR_VAPID_KEY_HERE

# Supabase (for image storage)
NEXT_PUBLIC_SUPABASE_URL=YOUR_SUPABASE_PROJECT_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY

# SMTP Configuration (for email notifications)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=your-app-password
```

**To get Firebase credentials:**
1. Go to [Firebase Console](https://console.firebase.google.com/) → Project Settings → Service Accounts
2. Click "Generate New Private Key" and save as `firebase-admin-key.json` in project root
3. Go to Project Settings → Cloud Messaging → Web Push Certificates
4. Generate a VAPID key pair and copy the public key to `NEXT_PUBLIC_FIREBASE_VAPID_KEY`

**To get Supabase credentials:**
1. Go to [Supabase Dashboard](https://supabase.com/dashboard) and create a new project
2. Go to Project Settings → API
3. Copy the Project URL to `NEXT_PUBLIC_SUPABASE_URL`
4. Copy the anon/public key to `NEXT_PUBLIC_SUPABASE_ANON_KEY`
5. Create a storage bucket named `payment-qr-codes` in the Supabase Storage section
6. Configure Storage RLS policies to allow public uploads:
   - Go to Storage → payment-qr-codes → Policies
   - Add a new policy with:
     - Policy name: "Allow public uploads"
     - Allowed operations: INSERT, SELECT
     - Target roles: anon, authenticated
     - Policy definition: `bucket_id = 'payment-qr-codes'`

### **3. Enabling Admin Features**
Security rules prevent client-side elevation to Admin. To enable admin controls for a user:
1. Log into the app to create your account.
2. Go to the **Firebase Console** > **Firestore**.
3. Create a collection named `admin_roles`.
4. Create a document where the **ID** is your **Auth UID**.
5. Add a field `uid` (string) with your UID as the value.

### **4. Session-Based Routing**
The app uses session-based routing where all features are scoped to an active session:
- **Session URLs**: All pages are under `/session/[id]` (e.g., `/session/ABCDEF/rankings`)
- **Shareable Links**: Session URLs can be shared directly with players
- **Session Persistence**: Sessions persist across page refreshes until ended by admin
- **Session Gate**: Players enter via 6-digit session codes generated by administrators

## ✨ Core Features

### **🏟️ Command Center (Dashboard)**
- **Real-Time Monitoring**: Live tracking of active matches and queued players.
- **Admin Hub**: Direct court assignment, manual matchmaking, and session termination.
- **Live Timers**: Visual match duration tracking for every active court.
- **Push Notifications**: Players receive notifications when assigned to courts (requires permission).

### **🤖 AI Commissioner (Matchmaking)**
- **Smart Pairing**: Uses Genkit AI and deterministic logic to balance skill levels and variety.
- **Quick Match**: One-click matchmaking that prioritizes waiting players while avoiding repeat partners.
- **Match Analysis**: Explainable AI that provides a 1-sentence justification for every suggested pairing.

### **🏆 Player Experience**
- **Skill Assessment**: 3-point diagnostic tool (Consistency, Movement, Matchplay) to determine skill tier (Level 1-7).
- **Session Gate**: Secure access via 6-digit session codes generated by administrators.
- **Live Leaderboard**: Automated rankings based on wins, win rate, and point differentials.
- **Push Notifications**: Real-time alerts for court assignments (Android, iOS 16.4+, Desktop).

### **💰 Financials & Payments**
- **Daily Split Calculator**: Admin tool to calculate per-player costs for shuttles, courts, and entry fees.
- **QR Payment Support**: Support for multiple payment methods (e.g., GCash, Maya) with easy QR scanning.
- **Verification Roster**: Real-time tracking of who has settled their daily club dues.

### **🔔 Push Notifications**
- **Firebase Cloud Messaging (FCM)**: Cross-platform push notifications for court assignments
- **Browser Support**: Chrome, Firefox, Safari 16.4+, Edge
- **PWA Ready**: Installable as progressive web app with background notifications
- **iOS Support**: Works on iOS 16.4+ when installed as PWA

## 🔧 Development

### **Installing Dependencies**
```bash
npm install
```

### **Running Development Server**
```bash
npm run dev
```

### **Building for Production**
```bash
npm run build
npm start
```

### **Firebase Deployment**
```bash
# Deploy to Firebase Hosting
firebase deploy

# Deploy specific services
firebase deploy --only firestore
firebase deploy --only functions
firebase deploy --only hosting
```

### **Local Development with Firebase Emulators**
The project includes Firebase emulator configuration in `firebase.json`:
- **Hosting**: Port 5000
- **Functions**: Port 5001
- **Firestore**: Port 8080
- **Database**: Port 9000
- **Auth**: Port 9099
- **Storage**: Port 9199

Start emulators:
```bash
firebase emulators:start
```

### **Testing Local Storage Sync**
The app uses a hybrid data layer with local storage as the primary source of truth and Firebase as the backend:
- All write operations immediately update local storage for fast UI
- Changes are synced to Firebase in the background
- Firebase real-time listeners keep local storage in sync with other clients
- Check browser DevTools > Application > Local Storage to see synced data

### **Testing Push Notifications**
1. Start the development server
2. Open the app in a supported browser
3. Click "Enable Notifications" in the Bench header
4. Grant notification permission
5. Assign a player to a court to trigger a test notification

**Note:** Ad blockers may block Firebase services. Disable them when testing notifications.
